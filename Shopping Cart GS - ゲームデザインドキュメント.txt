■ 本ドキュメントについて

本ドキュメントは「Shopping Cart GS」における唯一の正式仕様書である。
今後の設計、実装、バランス調整、マネタイズ設計は
本書の内容に従うものとする。

------------------------------------------------------------

1. 概要

タイトル:
Shopping Cart GS

ジャンル:
縦画面3Dランナー

対応プラットフォーム:
Android

ターゲット層:
・カジュアルプレイヤー
・8歳以上
・短時間プレイ志向ユーザー
・スコアアタック好き

コアコンセプト:
60秒間スーパーの通路を走り、
商品を集めてハイスコアを目指す。

高得点商品ほど重量があり、
重量が増えるほどスピードが低下する。

「欲張るか、走るか」のリスク設計を持つランナーゲーム。

------------------------------------------------------------

2. ゲームループ

1. ゲーム開始
2. 60秒間ラン
3. 商品収集
4. 重量増加 → 減速
5. タイムアップ
6. スコア計算
7. リザルト表示
8. リトライ

------------------------------------------------------------

3. 操作

・左右スワイプで3レーン移動
・商品は接触で自動取得

------------------------------------------------------------

4. プレイヤー仕様

レーン座標:
左   = -1.4
中央 = 0
右   = +1.4

スピードパラメータ:
BaseSpeed = 8.0
減速係数 = 0.15
最低速度 = 3.5

スピード計算式:
CurrentSpeed = BaseSpeed - (合計重量 × 減速係数)
CurrentSpeed = Max(CurrentSpeed, 最低速度)

------------------------------------------------------------

5. 商品システム

軽量:
ポイント = 10
重量 = 1
出現率 = 55%

中量:
ポイント = 30
重量 = 3
出現率 = 30%

重量:
ポイント = 80
重量 = 8
出現率 = 15%

出現間隔:
0〜20秒  = 1.2秒
20〜40秒 = 1.0秒
40〜60秒 = 0.8秒

------------------------------------------------------------

6. スコア計算

距離計算:
Distance += CurrentSpeed × DeltaTime

最終スコア:
FinalScore = Distance × (1 + 合計ポイント / 200)

設計意図:
・商品収集で倍率上昇
・Score の影響をやや抑え、Distance の価値を相対的に上げる
・heavy 一択になりにくいバランスへ寄せる
・重量増加でスピード低下
・リスクとリターンの両立

変更履歴
v1.0 初期仕様策定
【アイテムスクロール仕様】

・アイテムは SpawnRoot 配下に生成される
・スポーン後は -Z 方向へスクロールする
・移動速度はワールドスクロール速度と一致
・プレイヤー後方へ一定距離抜けたら削除
・Segmentとは独立管理とする
【アイテム移動仕様】
・スポーン後のアイテムは ItemMover により -Z 方向へ移動する
・移動速度は SegmentLoopScroller の現在速度を参照する
・プレイヤー後方へ一定距離抜けたアイテムは削除する
・タイムアップ時はアイテム移動も停止する
【アイテムスポーン座標仕様】
・アイテムのX座標は laneX の固定値（-1.4 / 0 / 1.4）をそのまま使用する
・スポーン座標は管理オブジェクトのTransformに依存しない
・Y座標は Inspector 調整可能な spawnY を使用する
・Z座標は playerTransform.position.z + spawnForwardOffset を使用する
・スポーン座標系は統一して管理する
【アイテム取得仕様】
・アイテムは Player との接触で取得される
・取得判定は Trigger を使用する
・取得時は itemName / weight / point をログ出力する
・取得後はアイテムを削除する
・重量加算およびポイント加算は次工程で実装する
【アイテム初期化仕様】
・スポーン時に ItemMover へ SegmentLoopScroller と Player Transform を設定する
・スポーン時に ItemPickup へ ItemDefinition を設定する
・生成後の clone は必ず初期化された状態で動作する
【アイテム取得判定仕様】
・アイテム取得は Trigger 判定を使用する
・Player または Item のいずれかに Rigidbody を持たせる
・Item 側 Collider は Is Trigger を ON とする
・Player タグ一致時に取得成功とする
【重量加算仕様】
・現在の合計重量を WeightManager で管理する
・ゲーム開始時の重量は 0 とする
・アイテム取得時に itemDefinition.weight を現在重量へ加算する
・重量は Game中 UI 表示する
・重量による減速反映は次工程で実装する
【重量減速仕様】
・現在速度は現在重量に応じて計算する
・計算式は CurrentSpeed = max(MinSpeed, BaseSpeed - CurrentWeight × SlowdownPerWeight)
・初期値は BaseSpeed = 12, SlowdownPerWeight = 0.15, MinSpeed = 3.5
・現在速度は SegmentLoopScroller に反映する
・距離計算も現在速度に基づく
・速度は Game中 UI 表示する
【ポイント加算仕様】
・現在の合計ポイントを ScoreManager で管理する
・ゲーム開始時のポイントは 0 とする
・アイテム取得時に itemDefinition.point を現在ポイントへ加算する
・ポイントは Game中 UI 表示する
・最終スコア計算は次工程で実装する
------------------------------------------------------------

【最終スコア計算仕様】
・最終スコアはタイムアップ時に計算する
・計算式は FinalScore = Distance × (1 + TotalScore / 200)
・Distance は累計移動距離を使用する
・TotalScore は累計ポイントを使用する
・FinalScore は FinalScoreManager で保持し、Result画面で参照する
------------------------------------------------------------

【Result表示仕様】
・タイムアップ時に Game シーン内の Result パネルを表示する
・Result パネルには Distance / Weight / Score / FinalScore を表示する
・Retry ボタンでゲームを初期状態から再開できる
・Result 表示中はゲーム進行を停止した状態とする
・Scene 遷移は現段階では行わない

【Resultリトライ仕様】
・タイムアップ時に ResultPanel を表示する
・Retry 実行時は Distance / Weight / Score / FinalScore を初期化する
・Retry 実行時は SpawnRoot 配下のアイテムを全削除する
・新規プレイはクリーンな状態で開始する

【Result表示項目】
・ResultPanel には Distance / Weight / FinalScore を表示する
・Score は内部計算用として保持するが、ResultPanel では表示しない
・タイムアップ時点の各管理値を表示する

【シーン遷移仕様】
・シーンは Title / Game の2種類を使用する
・Title シーンでは Start ボタンで Game シーンへ遷移する
・Game シーンでは初期状態で ResultPanel を非表示とする
・Retry は Game シーンを再開する
・TitleButton は Result から Title シーンへ戻る

------------------------------------------------------------

【UI実装方針】
・HUD と ResultPanel のレイアウトは Unity 上で手動調整する
・UI スクリプトは表示更新と表示切替のみ担当する
・RectTransform や配置座標はスクリプトで自動制御しない

【Resultボタン表示仕様】
・ResultPanel 表示時は RetryButton と TitleButton を両方表示する
・ResultPanel 非表示時は両ボタンも非表示状態とする
・TitleButton から Title シーンへ戻れる
【一時停止仕様】
・ゲーム中に Pause ボタンから一時停止できる
・Pause ボタンは HUD 右上に配置する
・Pause 中は PausePanel を表示する
・PausePanel には RESUME / RETRY / TOP ボタンを配置する
・PausePanel 表示中はゲーム画面を背後に残し、半透明の背景で暗く表示する
・RESUME ボタンで一時停止を解除し、ゲームへ戻る
・RETRY ボタンは既存の Retry 処理を使用する
・TOP ボタンは既存の Title 遷移処理を使用する
・Pause 中はタイマー、背景スクロール、アイテム移動、アイテムスポーン、プレイヤー操作、プレイヤーアニメーション、距離加算を停止する
・Pause 中はタイムアップしない
・ResultPanel 表示中は Pause できない
・Retry / TOP 実行時は一時停止状態を解除してから既存処理を実行する
・既存の3レーン移動、重量減速、スコア計算、Result仕様は変更しない
【奥背景棚 固定背景仕様】
・奥背景棚はスーパー通路の突き当たりに見える固定背景として扱う
・奥背景棚は Segment 配下ではなく StaticBackgroundRoot 配下に配置する
・奥背景棚は SegmentLoopScroller のスクロール対象に含めない
・奥背景棚は見た目専用であり、Collider / Rigidbody は追加しない
・SpawnRoot やアイテム生成・移動・取得・削除処理には関与させない
・背景表現上の調整であり、3レーン構造、重量減速、スコア計算、タイムアップ仕様は変更しない
・Segment配下の既存背景は維持し、奥背景棚のみ固定背景として例外的に分離する
【重量減速カーブ仕様】
・重量による減速は線形ではなくカーブ式を採用する
・計算式は
  CurrentSpeed = max(MinSpeed, BaseSpeed - pow(CurrentWeight, WeightCurvePower) × SlowdownPerWeight)
・初期値は BaseSpeed = 12, SlowdownPerWeight = 0.20, MinSpeed = 6.5, WeightCurvePower = 0.7
・前半の減速を緩やかにし、中盤以降で効く調整とする
【重量減速 暫定採用値】
・Base Speed = 12
・Slowdown Per Weight = 0.26
・Min Speed = 6.5
・Weight Curve Power = 0.7

【評価メモ】
・50秒前後で MinSpeed に到達
・現状の操作感は良好
・前半は操作しやすく、中盤以降で重さを感じ、終盤でしっかり負荷がかかる想定
・現段階ではこの値を暫定採用とする
・今後のアイテム価値バランス調整に応じて再調整の可能性あり
